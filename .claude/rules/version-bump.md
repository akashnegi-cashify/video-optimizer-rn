# Version Bump

When the user says **"increase version"**, **"bump version"**, **"update version to X.Y.Z"**, or similar, update **all** of the following files in a single pass:

## Files to update

| # | File | Field(s) | Example |
|---|---|---|---|
| 1 | `env/.env.stage` | `MARKETING_VERSION`, `CURRENT_PROJECT_VERSION` | `23.0.0`, `165` |
| 2 | `env/.env.stage.local` | same | same |
| 3 | `env/.env.beta` | same | same |
| 4 | `env/.env.beta.local` | same | same |
| 5 | `env/.env.prod` | same | same |
| 6 | `env/.env.prod.local` | same | same |
| 7 | `src/util/commonHeaders.util.ts` | fallback value in `getAppVersion()` | `'23.0.0'` |

## Rules

1. **`MARKETING_VERSION`** = versionName (e.g., `23.0.0`). Same across all env files. Drives iOS `CFBundleShortVersionString` via `tmp.xcconfig` (generated by `react-native-config`) → `SuperSale-{Debug,Release}.xcconfig` → `Info.plist` `$(MARKETING_VERSION)`. The same path covers both the main app and the `NotificationService` / `NotificationViewController` extensions. **Do NOT** add `MARKETING_VERSION = ...;` directly in `ios/SuperSale.xcodeproj/project.pbxproj` build settings — target-level pbxproj values override xcconfig and silently freeze the version.
2. **`CURRENT_PROJECT_VERSION`** = versionCode (e.g., `165`). Integer, same across all env files. Drives iOS `CFBundleVersion` via the same xcconfig chain. Same warning: do NOT hardcode in pbxproj.
3. **`commonHeaders.util.ts`** — update the fallback string in `getAppVersion()`.
4. If the user only says "increase version" without specifying numbers, ask which part to bump (major/minor/patch) and whether to increment the versionCode.
5. If the user provides only versionName (e.g., "update to 24.0.0"), auto-increment versionCode by 1 from the current value unless they specify otherwise.
